# Blog Images Placeholder Guide

Replace these placeholder files with actual images for the n8n AWS EC2 blog post.

## Required Images:

### 1. n8n-aws-featured.png
**Purpose:** Featured image for social media sharing (Open Graph/Twitter cards)
**Recommended size:** 1200x630px
**Description:** Eye-catching image showing n8n + AWS logos or a dashboard preview

### 2. n8n-dashboard.png
**Purpose:** Hero image at the start of the post
**Recommended size:** 800-1200px wide
**Description:** Screenshot of n8n dashboard showing workflows

### 3. ec2-launch.png
**Purpose:** Show AWS EC2 instance launch screen
**Recommended size:** 800-1000px wide
**Description:** Screenshot of AWS Console EC2 launch instance page

### 4. security-group.png
**Purpose:** Show security group configuration
**Recommended size:** 800-1000px wide
**Description:** Screenshot of AWS security group inbound rules with ports 22, 5678, 443

### 5. ssh-terminal.png
**Purpose:** Show SSH connection to EC2
**Recommended size:** 800-1000px wide
**Description:** Terminal screenshot showing SSH connection command and successful login

### 6. n8n-login.png
**Purpose:** Show n8n login screen
**Recommended size:** 800-1000px wide
**Description:** Screenshot of n8n login page at http://your-ip:5678

### 7. n8n-workflow-editor.png
**Purpose:** Show n8n workflow editor interface
**Recommended size:** 800-1200px wide
**Description:** Screenshot of n8n workflow editor with nodes

### 8. example-workflow.png
**Purpose:** Show a sample automation workflow
**Recommended size:** 800-1200px wide
**Description:** Screenshot of a complete n8n workflow example

## Tips:
- Use PNG format for screenshots with text
- Optimize images before uploading (use tools like TinyPNG)
- Keep file sizes under 500KB for faster loading
- Use descriptive alt text in the markdown
